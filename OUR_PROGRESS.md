## 2. WHERE WE STAND TODAY (baseline audit ŌĆö 2026-07-15)

**One-line verdict:** A working MVP of a **generic, single-shot DRS *plan generator***. It researches and writes a 15-stage rollout document for *any* company. It is **not Recykal-specific yet**, and it **executes nothing**.

### Architecture
- The **entire UI + all logic + state** lives in ONE file: `app/page.jsx` (~3,655 lines, `'use client'`). ~40 `useState` hooks, no components/Redux/context.
- **One workhorse API:** `POST /api/generate` ŌĆö builds a stage prompt ŌåÆ calls an LLM ŌåÆ parses JSON ŌåÆ saves. ~90% of the engine.
- **Chat assistant:** `POST /api/copilot` ŌĆö the "Binny" context-aware chat drawer.
- **Persistence:** browser talks **directly to Supabase** (`projects` table), with `localStorage` fallback. Server also logs every run to `drs_generation_runs` (also powers a Stage-2 "consensus" cache).
- **Prompts:** `lib/prompts.js` (giant switch, stages 3ŌĆō15) + `lib/prompts/stage2.js` + `lib/prompts/competitors.js` (stage 5).
- **LLM adapters:** `lib/llm/` ŌĆö Gemini (AI Studio), Vertex, Groq, Claude ŌĆö all behind a common `generateGrounded()` interface with a fallback chain + 240s timeout.
- **Default model:** Gemini 3.1 Pro (Vertex) ŌĆö `gemini-3.1-pro-preview`.

### The 15 stages
1. Setup (form, no LLM) ┬Ę 2. Geography Intel ┬Ę 3. Market Intel ┬Ę 4. Stakeholders ┬Ę 5. Competitors ┬Ę 6. Resistance ┬Ę 7. Narrative & Alignment ┬Ę 8. Blueprint ┬Ę 9. Execution ┬Ę 10. Launch Readiness ┬Ę 11. GTM Launch & Funnel ┬Ę 12. BTL Activation ┬Ę 13. Reputation Management ┬Ę 14. KPIs ┬Ę 15. Knowledge Base.
- Each stage = bespoke prompt ŌåÆ strict JSON ŌåÆ bespoke UI render block.
- **Grounded (real web search):** stages 2, 3, 4, 5, 6, 12. Everything else is ungrounded synthesis.
- **Dependency chain (loose, mostly optional):** 1ŌåÆall; 2ŌåÆ6,8,9,11,12,15; 3ŌåÆ4,6,14; 4ŌåÆ6,7; 8ŌåÆ12,15; 9ŌåÆ10.

### What's genuinely good (don't rebuild)
- Real grounded research on 6 stages (actually sourced, not hallucinated) ŌĆö our strongest asset.
- Multi-provider LLM fallback (rarely hard-crashes).
- Cascading sub-projects (national master ŌåÆ regional children inherit real numbers).
- Stage data chains forward (roadmap is somewhat coherent).
