# SMS Depth Map

Version: 0.1

## Depth Thesis

SMS becomes deep when it can repeatedly turn one enterprise sustainability solution into qualified pipeline, faster funnel movement, higher win probability, and reusable ROI proof.

The system must answer five questions:

1. What domain problem creates urgency?
2. Which accounts are most likely to buy?
3. Who is in the buying committee?
4. What message and proof does each role need?
5. Which funnel stage is blocking revenue?

## Depth By Workflow

| Workflow | Shallow Version | Deep Version |
|---|---|---|
| WF1 Domain Intelligence | Summarize the solution | Build regulation briefs, sector pain, use cases, claims, competitor view, and sales answers |
| WF2 Target Account Selection | Create a prospect list | Define ICP, tiers, TAM/SAM/SOM, fit score, trigger, and next action |
| WF3 Buying Committee Mapping | Find one contact | Map economic buyer, champion, blocker, procurement, CFO, compliance, and proof needs |
| WF4 Persona-Led Communication | Write generic copy | Create role-specific messages, proof, CTAs, objections, and sales talk tracks |
| WF5 TOFU Demand Creation | Run awareness campaigns | Reach ICP-fit accounts with thought leadership, events, reports, and retargeting |
| WF6 MOFU Evaluation Activation | Book meetings | Offer assessments, workshops, readiness audits, consultations, and pilot briefs |
| WF7 BOFU Conversion Enablement | Send proposals | Support ROI, procurement, references, executive briefings, and deal-specific proof |
| WF8 Funnel Velocity Optimization | Look at pipeline | Diagnose stage delays, rejection reasons, proposal aging, and experiment fixes |
| WF9 MQL to SQL to Closure Engine | Pass leads to sales | Define MQL/SQL, scoring, SLA, rejection reasons, conversion dashboard, and handoff loop |
| WF10 ROI And Proof Loop | Mention outcomes | Capture revenue, sustainability impact, case studies, references, and proof assets |

## Decision Gates

| Gate | Question | Required Evidence | Decision |
|---|---|---|---|
| Solution Selection | Is this solution a priority? | Revenue potential, urgency, proof, SME capacity | Select / defer |
| ICP Approval | Do we know who should buy? | ICP definition, sectors, triggers, disqualifiers | Approve / refine |
| Campaign Activation | Are messages and assets ready? | Persona matrix, TOFU offer, target accounts | Launch / revise |
| MQL Acceptance | Should Sales accept these leads? | MQL criteria, fit score, engagement, trigger | Accept / reject |
| SQL Qualification | Is there real buying intent? | Problem, timeline, committee path, next step | Qualify / nurture |
| Proposal | Is the opportunity ready for commercial proposal? | Scope, ROI, stakeholders, procurement path | Propose / discover more |
| Executive Support | Is the deal strategic or stuck? | Deal value, blocker, stage age, executive need | Escalate / continue |
| Proof Creation | Is there a usable outcome? | Customer result, approval, metrics | Create proof / wait |

## Data Depth

Leading indicators:

- Target accounts identified
- Engaged accounts
- MQLs generated
- MQL to SQL conversion
- SQLs accepted
- Buying committee completeness
- Proposal turnaround time

Lagging indicators:

- Opportunities created
- Pipeline value
- Pipeline coverage
- Opportunity to win rate
- Average deal size
- Revenue sourced
- Revenue influenced
- CAC payback

Feedback loops:

- Sales rejection reasons update ICP and lead scoring.
- Objections update persona messaging and proof assets.
- Stage delays update nurture, proposal, and executive support workflows.
- Closed-won outcomes update case studies, ROI proof, and campaigns.

## Capability Depth

Depth 1:

- SMS architecture exists.
- Workflow library and templates exist.

Depth 2:

- One solution is selected.
- Solution instantiation brief is completed.
- Domain, ICP, account, and persona assets are drafted.

Depth 3:

- Weekly SMS revenue standup runs.
- MQL/SQL handoff is tracked.
- Funnel velocity is visible.

Depth 4:

- Rejection reasons, objections, and stage delays improve targeting, messaging, and proof.
- ROI proof improves TOFU, MOFU, and BOFU conversion.

Depth 5:

- New enterprise solutions can be launched by cloning the system.
- Proof from one solution improves credibility across the portfolio.
