# SMS 30-Day Build Backlog

Version: 0.1

## Build Principle

Build the smallest version of SMS that can prove the domain-account-persona-funnel-revenue loop for one solution.

Recommended first solution: EPR Compliance or DRS Readiness Consulting.

## Week 1: Solution Setup

Goal: define the solution and create the minimum SMS operating spine.

Tasks:

- Select first enterprise sustainability solution.
- Complete solution instantiation template.
- Draft solution thesis.
- Define ICP.
- Define target sectors.
- Define Enterprise Sustainability Revenue Index baseline.
- Create first weekly SMS dashboard.

Outputs:

- Completed solution instantiation brief.
- Solution thesis.
- ICP definition.
- KPI baseline.
- Weekly dashboard v1.

## Week 2: Domain And Account Intelligence

Goal: create the expertise and account universe.

Tasks:

- Draft domain playbook.
- Draft regulation or trigger brief.
- Build target account list.
- Segment accounts into Tier 1, Tier 2, and Tier 3.
- Estimate TAM/SAM/SOM.
- Build account scoring model.

Outputs:

- Domain playbook v1.
- Regulation/trigger brief.
- Target account list.
- Account scoring model.
- TAM/SAM/SOM estimate.

## Week 3: Buying Committee And Messaging

Goal: create persona-level relevance for target accounts.

Tasks:

- Build buying committee map.
- Build persona messaging matrix.
- Draft objection handling guide.
- Draft email sequence.
- Draft TOFU campaign brief.
- Draft MOFU assessment or workshop offer.

Outputs:

- Buying committee map.
- Persona messaging matrix.
- Objection handling guide.
- Email sequence.
- TOFU campaign brief.
- MOFU offer.

## Week 4: Funnel And Proof Engine

Goal: activate handoff, conversion, and proof systems.

Tasks:

- Define MQL and SQL criteria.
- Build lead scoring model.
- Build handoff SLA.
- Draft proposal template.
- Draft ROI calculator.
- Build funnel velocity dashboard.
- Run first weekly SMS revenue standup.

Outputs:

- MQL/SQL definition.
- Lead scoring model.
- Handoff SLA.
- Proposal template.
- ROI calculator.
- Funnel dashboard.
- SMS v0.2 update list.

## Definition Of Done For First 30 Days

- One solution has a completed instantiation brief.
- ICP and target account list exist.
- Domain playbook exists.
- Buying committee map exists.
- Persona messaging matrix exists.
- TOFU and MOFU offers exist.
- MQL/SQL definitions exist.
- Proposal and ROI templates exist.
- Weekly dashboard is operating.
- The biggest funnel bottleneck is known.

## Decisions Needed From Leadership

- Which solution launches first?
- Which sectors are in scope?
- What counts as an MQL and SQL?
- What pipeline target should the first 30 days support?
- Which proof assets already exist?
- Which solution consultants are available for MOFU and BOFU support?
